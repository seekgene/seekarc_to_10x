# SeekARC_to_10X: SeekARC to 10x Genomics Conversion Pipeline

This pipeline is engineered to bridge the compatibility gap between the SeekARC sequencing platform and the 10x Genomics downstream analysis ecosystem. It efficiently processes raw sequencing data from both SeekARC Multiome (RNA+ATAC) and Single ATAC experiments. The core workflow integrates automated multi-sample aggregation, high-precision quality control via Fastp, and rigorous barcode mapping and format reconstruction algorithms. This ensures that the output data—including file structure, sequence format, and metadata—fully complies with Cell Ranger ARC and Cell Ranger ATAC input standards, enabling seamless integration into mainstream single-cell multi-omics analysis workflows.

## Overview

The SeekARC platform produces data that requires specific preprocessing to be compatible with the widely used 10x Genomics analysis ecosystem. This pipeline bridges that gap by:

1.  **Preprocessing**: Handling multiple input files (e.g., multiple lanes) and merging them automatically.
2.  **Quality Control**: Using `fastp` to trim adapters and filter low-quality reads.
3.  **Format Conversion**:
    *   Mapping SeekARC barcodes to 10x Genomics whitelist barcodes.
    *   Restructuring Fastq files to match Cell Ranger input specifications.
    *   (Multiome) Removing Template Switch Oligo (TSO) sequences from RNA reads.

## Features

*   **Dual Mode Support**:
    *   **Multiome Mode (`--mode multiome`)**: Processes paired RNA-seq and ATAC-seq data.
    *   **Single ATAC Mode (`--mode atac`)**: Processes ATAC-seq data only.
*   **Robust Input Handling**: Supports complex input scenarios, such as data split across multiple lanes or files. Simply provide all file paths, and the pipeline handles merging.
*   **Integrated Quality Control**: comprehensive QC reports (HTML/JSON) generated by `fastp` for every sample.

## Installation & Environment

The pipeline relies on a specific Python environment and several external tools.

### 1. Conda Environment

We recommend using Conda to manage dependencies. A `env.yaml` file is provided to create the environment.

```bash

# Activate the environment

```

*   Python >= 3.10
*   cutadapt
*   click
*   dnaio
*   xopen
*   pigz

### 2. External Tools

Ensure the following tools are installed and their paths are correctly configured in `pipeline.py` (or accessible via `$PATH`):

*   **Fastp**: Included in the `bin/` directory of this repository.
*   **conv.0.1.1**: Included in the `bin/` directory of this repository.

## Usage

Run the pipeline using the `python` command. Below is the full list of arguments:

```bash
python pipeline.py \
    --mode <multiome|atac> \
    --outdir <OUTPUT_DIRECTORY> \
    --sample <SAMPLE_NAME> \
    --rna_r1 <RNA_R1_FILES> [RNA_R1_FILES ...] \
    --rna_r2 <RNA_R2_FILES> [RNA_R2_FILES ...] \
    --atac_r1 <ATAC_R1_FILES> [ATAC_R1_FILES ...] \
    --atac_r2 <ATAC_R2_FILES> [ATAC_R2_FILES ...] \
    --core <THREADS>
```

### Arguments

#### Required Arguments

| Argument | Description |
| :--- | :--- |
| `--outdir` | The main directory where all outputs (QC, intermediate files, and converted sequencing files) will be saved. |
| `--sample` | A unique identifier for the sample. This name will be used for output files. |

#### Pipeline Mode

| Argument | Default | Description |
| :--- | :--- | :--- |
| `--mode` | `multiome` | Specify the analysis mode: `multiome` (RNA+ATAC) or `atac` (ATAC only). |

#### Input Files

You can provide multiple files for each argument (separated by spaces). The pipeline will automatically merge them.

**For Multiome Mode (`--mode multiome`):**
*   `--rna_r1`: Path(s) to RNA Read 1 files.
*   `--rna_r2`: Path(s) to RNA Read 2 files.
*   `--atac_r1`: Path(s) to ATAC Read 1 files.
*   `--atac_r2`: Path(s) to ATAC Read 2 files.

**For Single ATAC Mode (`--mode atac`):**
*   `--atac_r1`: Path(s) to ATAC Read 1 files.
*   `--atac_r2`: Path(s) to ATAC Read 2 files.
*   *(RNA arguments are ignored in this mode)*

#### Resource Options

| Argument | Default | Description |
| :--- | :--- | :--- |
| `--core` | `12` | Number of CPU threads to use for parallel processing (Fastp, conversion). |

## Workflow Detail

### Step 1: Input Preparation
The pipeline first verifies all input files. If multiple files are provided for a single read type (e.g., `L001_R1.fq.gz`, `L002_R1.fq.gz`), they are concatenated into a single file in the `{outdir}/merged/` directory.

### Step 2: Quality Control (Fastp)
Raw reads are processed with `fastp`.
*   **Trimming**: Adapters and low-quality bases (tail trimming) are removed.
*   **Reporting**: HTML and JSON reports are generated in `{outdir}/fastp/` for inspection.
*   **Output**: Cleaned Fastq files are saved to `{outdir}/fastp/` for the next step.

### Step 3: Barcode Translation & Conversion
This is the core conversion step.
*   **Whitelist Generation**: Valid barcodes are extracted from the provided 10x whitelist files.
*   **Mapping**:
    *   **Multiome**: A mapping file (`dd_rna_atac_map.tsv`) is created to link SeekARC barcodes to their 10x counterparts. This file contains three columns (SeekARC Barcode, 10x RNA Barcode, 10x ATAC Barcode) as 10x RNA and ATAC libraries use different barcodes.
    *   **ATAC**: A direct mapping is performed using the ATAC whitelist. A `map.tsv` file is generated in `{outdir}/data/`.
*   **File Conversion**: The `conv.0.1.1` tool and custom Python scripts perform the following operations to ensure full Cell Ranger compatibility:
    *   **Barcode Correction**: Corrects sequencing errors based on the 10x whitelist (allowing 1 bp mismatch).
    *   **Sequence Trimming**: Removes adapter sequences and TSO (Template Switch Oligo) sequences (Multiome RNA only).
    *   **Reformatting**: Transforms Fastq headers and sequence structures.

### Step 4: Cell Ranger Analysis (Manual)
The pipeline generates the necessary input files (`library.csv` for Multiome, processed Fastq files for ATAC) but **does not** execute Cell Ranger automatically. You must run Cell Ranger manually using the commands provided in the "Running Cell Ranger" section below.

## Output Structure

The output directory is organized as follows:

```text
{outdir}/
├── data/                         # Intermediate Converted Data
│   ├── A/                        # ATAC converted files
│   ├── E/                        # RNA converted files (Multiome only)
│   ├── map.tsv                   # Barcode mapping file (ATAC mode)
│   ├── dd_rna_atac_map.tsv     # Multiome Barcode mapping file
│   └── ...
├── fastp/                        # Quality Control Results
│   ├── {sample}_atac_fastp.html  # QC report for ATAC reads
│   ├── {sample}_rna_fastp.html   # QC report for RNA reads
│   └── ...
├── merged/                       # Merged Input Files
│   └── ... (only if multiple input files were provided)
└── library.csv                   # Cell Ranger Library CSV (Multiome only)
```

## Running Cell Ranger

After the pipeline completes successfully, you can run Cell Ranger using the generated files.

### 1. Multiome (Cell Ranger ARC)

For Multiome mode, the pipeline generates a `library.csv` file in your output directory.

```bash
# Example command
/path/to/cellranger-arc count \
    --id=<SAMPLE_NAME> \
    --reference=/path/to/refdata-cellranger-arc-GRCh38-2020-A-2.0.0 \
    --libraries=<OUTPUT_DIRECTORY>/library.csv \
    --localcores=12 \
    --localmem=64
```

### 2. Single ATAC (Cell Ranger ATAC)

For Single ATAC mode, use the converted Fastq files located in `<OUTPUT_DIRECTORY>/data/A/step1/`.

```bash
# Example command
/path/to/cellranger-atac count \
    --id=<SAMPLE_NAME> \
    --reference=/path/to/refdata-cellranger-arc-GRCh38-2020-A-2.0.0 \
    --fastqs=<OUTPUT_DIRECTORY>/data/A/step1/ \
    --chemistry=ARC-v1 \
    --sample=<SAMPLE_NAME> \
    --localcores=12 \
    --localmem=64
```
